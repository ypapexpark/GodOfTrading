# GodOfTrading 원칙 카탈로그 v1

**목적:** 유명 트레이더/문헌의 *스토리*가 아니라, **검증 가능한 규칙**만 봇에 옮긴다.  
**파이프라인:** 원칙 → 정량 가설 → 반사실 → 관찰(paper) → 라이브.  
**한 번에 실험 1~2개.** 동시 다중 변경 금지.

작성: 2026-07-11 · 상태: v1 (레짐 라우터 구현과 동시 채택)

---

## 0. 가드레일 (명성 규칙)

| ID | 규칙 |
|----|------|
| G0 | 출처 없는 “고수가 그랬대”로 라이브 on 금지 |
| G1 | 반사실 EV≤0 또는 표본 부족이면 폐기/paper |
| G2 | 기존 양의 엣지(EMA+돌파/거래량급등)를 해치면 채택 금지 |
| G3 | 한 달 라이브 실험 ≤ 2개 원칙 |
| G4 | 목표 함수 = 기대값·DD·빈도 (승률 단독 최적화 금지) |

## 1. 원칙 목록

### P1. 시장은 국면이 있다 (Markets have regimes)
- **출처 계열:** 추세추종 공통(터틀·추세문헌), 평균회귀 문헌, 실전 인터뷰 공통
- **의미:** 추세 전략과 역추세/평균회귀를 같은 파라미터로 돌리지 않는다
- **시스템 매핑:** `regime.py` 레짐 라우터
- **상태:** **라이브 v1** (2026-07-11)

### P2. 생존이 알파보다 앞선다
- **출처 계열:** 거의 모든 프로 리스크 문헌
- **매핑:** DD 하드스톱, 일손실 softcap, 포트폴리오 캡, Binance 단건 손실 캡
- **상태:** 라이브 유지

### P3. 추세 순응 · 눌림목 (Pullback with trend)
- **출처 계열:** 추세 순응 공통
- **매핑:** `EMA눌림목(+거래량/돌파)`, MTF·EMA 정렬
- **상태:** 주력 라이브 (실측 +EV 후보)

### P4. 추세 재개 (Continuation / hidden)
- **출처 계열:** 다이버전스·추세 지속 문헌
- **매핑:** `hidden_*`, 신선도·SL% 캡
- **상태:** 라이브 (상위TF 위주; 15m 제외)

### P5. 평균회귀는 횡보/극단에서만
- **출처 계열:** Connors RSI(2), VWAP 회귀 문헌
- **매핑:** RSI2·VWAP·일반 bullish/bearish → 관찰 paper
- **상태:** paper (레짐=range일 때만 향후 재평가)

### P6. 과열 추격 금지
- **출처 계열:** “모멘텀 쫓지 마라” 공통
- **매핑:** `EXTENSION_HARD_BLOCK_PCT`, VWAP 추격 차단
- **상태:** 라이브

### P7. 손절 = 논리 무효 지점
- **출처 계열:** 구조 손절 공통
- **매핑:** ATR/피봇 SL, `MAX_ENTRY_SL_PCT`
- **상태:** 라이브

### P8. 익절 분할 + 러너
- **출처 계열:** 스케일 아웃·트레일 공통
- **매핑:** 분할 TP, pre-TP 보호, TP1 후 +R 잠금, 전 포지션 트레일
- **상태:** 라이브 (2026-07-11 강화)

### P9. 숏은 롱과 대칭이 아니다
- **출처 계열:** 크립토 상승 편향·공매도 스퀴즈 실무
- **매핑:** SHORT MTF/EMA 필수, 전역 사이즈×0.5, ROI 레버 보정 금지
- **상태:** 라이브 (실측 R:R 0.38 대응)

### P10. 선택성 (A+ only)
- **출처 계열:** “매일 매매하지 않는다”
- **매핑:** 화이트리스트, 15m EMA만, 관찰모드 paper, VERY STRONG+
- **상태:** 라이브

### P11. 1R 사고 (포지션 = 위험 단위)
- **출처 계열:** 밴 타프·리스크 문헌 공통
- **매핑:** risk% 사이징, Binance max margin/SL loss cap
- **상태:** 라이브

### P12. 유동성·체결 품질
- **출처 계열:** 실행 품질 공통
- **매핑:** 레이더 TopN, 최소 거래대금, 스프레드( BTC sync )
- **상태:** 부분 — 향후 세션/스프레드 강화

## 4. 고수/유명인 지표·방법 정책 (중요)

**원칙은 쓰고, “그 사람 전용 지표 복붙”은 안 쓴다.**

| 쓴다 | 안 쓴다 |
|------|---------|
| 여러 고수/문헌에 **공통**인 원칙 (레짐, 생존, 1R, 과열 금지, 분할익절) | “○○ 고수가 쓰는 비밀 지표” 단발 추가 |
| 이미 엔진에 있는 지표의 **용도 정리** (MACD=다이버전스/히스토그램 soft) | RSI+Stoch+CCI+MACD에 **또 다른 오실레이터** 적층 |
| 반사실·실거래로 검증된 필터 | 인터뷰 문장 → 즉시 라이브 엔트리 |
| 관찰모드 paper 후 졸업 | 책 제목만 있는 전략 이름 붙이기 |

**이미 반영된 “고수 공통”:** P1 레짐 라우터, P2 생존, P8 러너, P9 SHORT 비대칭, P10 선택성.  
**문헌 격차로 코드에만 있는 것(관찰 paper):** RSI2, VWAP회귀, 파라볼릭 — **아직 주력 아님**.

주간 리포트(`tools/weekly_learning_report.py`)는 **제안만** 하고 config는 사람 승인 후 변경.
